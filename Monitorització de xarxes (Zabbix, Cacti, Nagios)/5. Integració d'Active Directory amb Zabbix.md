# Guia Detallada de Configuració d'Active Directory i Integració amb Zabbix

---

## 1. Instal·lació del Rol de Servidor
El primer pas és preparar el servidor Windows Server per actuar com a controlador de domini.

### Instal·lació del Rol AD DS
![Instal·lació ROL](/img/AD/Instal·laciò ROL.png)
En aquesta pantalla del gestor del servidor, seleccionem el rol **"Servicios de dominio de Active Directory"**. Aquest component és el que permetrà gestionar usuaris, equips i polítiques de seguretat de manera centralitzada.

---

## 2. Configuració del Controlador de Domini (Pas a Pas)
Un cop instal·lat el rol, procedim a promocionar el servidor a controlador de domini.

### DC1: Selecció de l'operació
![DC1](/img/AD/DC1.png)
Triem l'opció **"Agregar un nuevo bosque"** i definim el nom del domini arrel com a `aos.cat`. Aquesta és la base de la nostra estructura de directori.

### DC2: Opcions del controlador
![DC2](/img/AD/DC2.png)
Establim els nivells funcionals del bosc i del domini (Windows Server 2016). També definim la **contrasenya de DSRM** (Modo de restauración de servicios de directorio), vital per a la recuperació del sistema.

### DC3: Opcions de DNS
![DC3](/img/AD/DC3.png)
Configuració de les opcions de DNS. Apareix un avís sobre la delegació, que és normal en aquest punt de la instal·lació inicial.

### DC4: Nom NetBIOS
![DC4](/img/AD/DC4.png)
El sistema assigna automàticament el nom NetBIOS **AOS**. Aquest nom s'utilitza per a la compatibilitat amb versions anteriors de Windows.

### DC5: Rutes d'accés
![DC5](/img/AD/DC5.png)
Definim on s'emmagatzemaran la base de dades (NTDS), els fitxers de registre i la carpeta **SYSVOL**. S'han deixat les rutes per defecte de Windows.

### DC6: Revisió d'opcions
![DC6](/img/AD/DC6.png)
Resum de totes les configuracions seleccionades. És el moment de verificar que el domini, el NetBIOS i els nivells funcionals són corrects.

### DC7: Comprovació de requisits
![DC7](/img/AD/DC7.png)
L'assistent verifica que el servidor compleix tots els requisits. Un cop apareix el missatge verd de **"Todas las comprobaciones se realizaron correctamente"**, podem procedir.

### DC8: Instal·lació en curs
![DC8](/img/AD/DC8.png)
Procés d'aplicació de les configuracions i creació del directori.

### DC9: Reinici del sistema
![DC9](/img/AD/DC9.png)
El servidor necessita reiniciar-se per aplicar els canvis i convertir-se oficialment en un controlador de domini.

### DC10: Inici de sessió al domini
![DC10](/img/AD/DC10.png)
Primer inici de sessió un cop el domini està actiu. Observem que ara ens identifiquem com a **AOS\Administrador**.

### DC11: Usuaris i equips d'AD
![DC11](/img/AD/DC11.png)
Obertura de la consola principal de gestió: **Usuarios y equipos de Active Directory**.

### DC12: Estructura del domini
![DC12](/img/AD/DC12.png)
Vista inicial de l'arbre del domini `aos.cat` amb les seves carpetes per defecte (Builtin, Computers, Users, etc.).

### DC13: Nova Unitat Organitzativa
![DC13](/img/AD/DC13.png)
Creació d'una **Unitat Organitzativa (UO)** anomenada **Zabbix** per mantenir els objectes relacionats amb la monitorització ben organitzats.

### DC14: Creació d'usuari de monitorització
![DC14](/img/AD/DC14.png)
Dins de l'UO Zabbix, creem un nou usuari anomenat **zabbix.monitor**. Aquest usuari serà l'encarregat de fer les consultes LDAP des de Zabbix.

### DC15: Contrasenya de l'usuari
![DC15](/img/AD/DC15.png)
Assignem una contrasenya segura i marquem l'opció **"La contraseña nunca expira"** per evitar talls en el servei de monitorització.

### DC16: Resum de l'usuari
![DC16](/img/AD/DC16.png)
Confirmació final de les dades de l'usuari `zabbix.monitor@aos.cat`.

### DC17: Usuari llistat
![DC17](/img/AD/DC17.png)
L'usuari apareix correctament dins de l'UO Zabbix a la consola de gestió.

---

## 3. Integració amb Zabbix i Resolució d'Errors

### DC18: Configuració LDAP a Zabbix
![DC18](/img/AD/DC18.png)
Anem a l'apartat d'autenticació de Zabbix i intentem activar la **Configuració LDAP**.

### L'Error de PHP LDAP
![Error](/img/AD/Error.png)
En intentar activar-lo, ens trobem amb un error: **"Extensión PHP LDAP faltante"**. El servidor web on corre Zabbix no té el mòdul necessari per parlar amb l'Active Directory.

### Solució - Part 1
![Solució 1](/img/AD/Comanda_solucio1.png)
Executem la comanda d'instal·lació del paquet `php-ldap` al servidor Linux de Zabbix.

### Solució - Part 2
![Solució 2](/img/AD/Comandes Solucio2.png)
Reiniciem els serveis (Apache o PHP-FPM) perquè els canvis tinguin efecte i el mòdul es carregui.

### DC19: Activació LDAP
![DC19](/img/AD/DC19.png)
Un cop instal·lada l'extensió, ja podem marcar la casella **"Activar autenticación LDAP"**.

### DC20: Configuració del servidor LDAP
![DC20](/img/AD/DC20.png)
Introduïm les dades del nostre DC:
- **IP**: 192.168.56.102
- **DN Base**: `DC=aos,DC=cat`
- **DN del enlace**: `zabbix@aos.cat` (l'usuari que hem creat abans).

### DC21: Prova d'autenticació
![DC21](/img/AD/DC21.png)
Abans de guardar, fem una prova d'inici de sessió amb l'usuari `zabbix` per verificar que la comunicació és correcta.

### DC23: Èxit en la prova
![DC23](/img/AD/DC23.png)
Zabbix ens confirma: **"Inicio de sesión correcto"**. La integració LDAP funciona.

### DC24: Servidor guardat
![DC24](/img/AD/DC24.png)
El servidor LDAP queda registrat i actiu a la llista de servidors d'autenticació.

### DC25: Alta d'usuari a Zabbix
![DC25](/img/AD/DC25.png)
Creem l'usuari `zabbix.monitor` dins de la base de dades de Zabbix, però vinculat a l'autenticació externa.

### DC26: Assignació de permisos
![DC26](/img/AD/DC26.png)
Li assignem el **"Super admin role"** perquè pugui gestionar totes les funcions del panell de monitorització.

### DC27: Llista d'usuaris final
![DC27](/img/AD/DC27.png)
L'usuari ja apareix a la llista de Zabbix amb el mètode d'autenticació correcte.

### DC28: Login amb usuari LDAP
![DC28](/img/AD/DC28.png)
Fem la prova real de login a la pantalla d'entrada de Zabbix amb les credencials de l'Active Directory.

### DC29: Panell de control
![DC29](/img/AD/DC29.png)
Accés concedit. Ja estem dins de Zabbix amb l'usuari del domini.

### DC30: Monitorització activa
![DC30](/img/AD/DC30.png)
El sistema ja està monitoritzant el servidor **DC01** i altres màquines.

### DC31: Detalls de rendiment
![DC31](/img/AD/DC31.png)
Vista dels gràfics de càrrega de CPU i ús de memòria del controlador de domini.

### DC32: Estat Global
![DC32](/img/AD/DC32.png)
Pantalla final amb el resum de l'estat del sistema, confirmant que tot funciona sense problemes.
