# Integració d'Active Directory amb Zabbix

Aquest document presenta el pas a pas complet del procés d'instal·lació, configuració i integració realitzat, amb una explicació individual per a cada captura de pantalla.

---

## 1. Instal·lació del Rol de Servidor
El primer pas és preparar el servidor Windows Server per actuar com a controlador de domini.

### Instal·lació del Rol AD DS
![Instal·lació ROL](img/AD/Instal·laciò_ROL.png)

En aquesta pantalla del gestor del servidor, seleccionem el rol **"Servicios de dominio de Active Directory"**. Aquest component és el que permetrà gestionar usuaris, equips i polítiques de seguretat de manera centralitzada.

---

## 2. Configuració del Controlador de Domini (Pas a Pas)
Un cop instal·lat el rol, procedim a promocionar el servidor a controlador de domini.

### Pas 1: Selecció de l'operació
![Pas 1](img/AD/DC1.png)

Triem l'opció **"Agregar un nuevo bosque"** i definim el nom del domini arrel com a `aos.cat`. Aquesta és la base de la nostra estructura de directori.

### Pas 2: Opcions del controlador
![Pas 2](img/AD/DC2.png)

Establim els nivells funcionals del bosc i del domini (Windows Server 2016). També definim la **contrasenya de DSRM** (Modo de restauración de servicios de directorio), vital per a la recuperació del sistema en cas de fallada.

### Pas 3: Opcions addicionals (NetBIOS)
![Pas 3](img/AD/DC3.png)

El sistema assigna automàticament el nom NetBIOS **AOS**. Aquest nom s'utilitza per a la compatibilitat amb versions anteriors de Windows i per a la identificació a la xarxa local.

### Pas 4: Rutes d'accés
![Pas 4](img/AD/DC4.png)

Definim on s'emmagatzemaran la base de dades (NTDS), els fitxers de registre i la carpeta **SYSVOL**. S'han deixat les rutes per defecte de Windows per a una configuració estàndard.

### Pas 5: Revisió d'opcions
![Pas 5](img/AD/DC5.png)

Resum de totes les configuracions seleccionades. És el moment de verificar que el domini, el NetBIOS i els nivells funcionals són els que hem definit abans de començar la instal·lació real.

### Pas 6: Comprovació de requisits
![Pas 6](img/AD/DC6.png)

L'assistent verifica que el servidor compleix tots els requisits previs. Un cop apareix el missatge verd de **"Todas las comprobaciones se realizaron correctamente"**, podem procedir amb la instal·lació.

### Pas 7: Instal·lació completada
![Pas 7](img/AD/DC7.png)

Missatge de confirmació on s'indica que el servidor s'ha configurat correctament com a controlador de domini i es reiniciarà per aplicar els canvis.

### Pas 8: Administració d'Usuaris i Equips
![Pas 8](img/AD/DC8.png)

Un cop reiniciat, obrim la consola **Usuarios y equipos de Active Directory** per gestionar el nostre nou domini `aos.cat`.

### Pas 9: Creació de la Unitat Organitzativa
![Pas 9](img/AD/DC9.png)

Creem una nova **Unitat Organitzativa (UO)** anomenada **Zabbix** per tenir localitzats tots els usuaris que utilitzarem per a la monitorització.

### Pas 10: Creació de l'usuari zabbix.monitor
![Pas 10](img/AD/DC17.png)

Dins de l'UO Zabbix, creem l'usuari **zabbix.monitor**. Aquest usuari serà l'encarregat de fer les consultes LDAP des de la plataforma Zabbix.

### Pas 11: Configuració de la contrasenya
![Pas 11](img/AD/DC18.png)

Assignem la contrasenya i marquem l'opció **"La contraseña nunca expira"** per evitar talls de servei.

---

## 3. Integració amb Zabbix i Resolució d'Errors

### Pas 12: Menú d'Autenticació a Zabbix
![Pas 12](img/AD/DC13.png)

Anem a Zabbix i entrem al menú d'Autenticació per configurar el servidor LDAP.

### L'Error de PHP LDAP
![Error](img/AD/Error.png)

En intentar configurar LDAP, apareix l'error **"Extensión PHP LDAP faltante"**, ja que el servidor web no té el mòdul necessari.

### Solució - Part 1: Instal·lació del mòdul
![Solució 1](img/AD/Comanda_solucio1.png)

Executem la comanda `apt install php-ldap` al servidor de Zabbix.

### Solució - Part 2: Reinici de serveis
![Solució 2](img/AD/Comandes_Solucio2.png)

Reiniciem el servidor web perquè els canvis tinguin efecte.

### Pas 13: Configuració del Servidor LDAP
![Pas 13](img/AD/DC14.png)

Introduïm la IP del DC, el DN base (`DC=aos,DC=cat`) i l'usuari d'enllaç (`zabbix@aos.cat`).

### Pas 14: Prova de connexió
![Pas 14](img/AD/DC15.png)

Realitzem una prova d'inici de sessió per verificar que Zabbix pot parlar amb l'Active Directory.

### Pas 15: Connexió correcta
![Pas 15](img/AD/DC16.png)

Zabbix ens confirma que l'inici de sessió ha estat correcte.

### Pas 16: Alta de l'usuari zabbix.monitor a Zabbix
![Pas 16](img/AD/DC20.png)

Creem l'usuari dins de Zabbix i el vinculem a l'autenticació LDAP.

### Pas 17: Assignació del Rol Super Admin
![Pas 17](img/AD/DC21.png)

Li assignem el rol de **Super administrador** per tenir control total sobre la monitorització.

### Pas 18: Confirmació de l'alta
![Pas 18](img/AD/DC22.png)

Fem clic a "Agregar" per finalitzar la creació de l'usuari a Zabbix.

### Pas 19: Llista d'usuaris amb zabbix.monitor
![Pas 19](img/AD/DC23.png)

L'usuari ja apareix actiu a la llista d'usuaris de Zabbix.

### Pas 20: Login amb zabbix.monitor
![Pas 20](img/AD/DC24.png)

Provem d'entrar a Zabbix amb les credencials del domini.

### Pas 21: Dashboard de zabbix.monitor
![Pas 21](img/AD/DC25.png)

Dashboard operatiu un cop hem entrat amb l'usuari de l'AD.

---

## 4. Configuració de l'usuari zabbix.suport
Per verificar la gestió de rols, creem un segon usuari amb permisos restringits.

### Pas 22: Creació de zabbix.suport a l'AD
![Pas 22](img/AD/DC26.png)

Creem un segon usuari anomenat **zabbix.suport** dins de la mateixa UO de l'Active Directory.

### Pas 23: Contrasenya de zabbix.suport
![Pas 23](img/AD/DC27.png)

Assignem la contrasenya corresponent al nou usuari de suport.

### Pas 24: Alta de zabbix.suport a Zabbix
![Pas 24](img/AD/DC28.png)

Donem d'alta l'usuari a Zabbix utilitzant el mètode d'autenticació LDAP.

### Pas 25: Assignació del Rol User (Restringit)
![Pas 25](img/AD/DC29.png)

En aquest cas, li assignem el **User role** per limitar el seu accés a la configuració del sistema.

### Pas 26: Confirmació de l'usuari de suport
![Pas 26](img/AD/DC30.png)

Guardem els canvis per activar l'usuari de suport a la plataforma.

### Pas 27: Login amb zabbix.suport
![Pas 27](img/AD/DC31.png)

Iniciem sessió amb el compte de suport per verificar que l'autenticació funciona.

### Pas 28: Dashboard restringit de zabbix.suport
![Pas 28](img/AD/DC32.png)

Podem observar que l'usuari de suport té un Dashboard més limitat, confirmant que els rols s'apliquen correctament.
