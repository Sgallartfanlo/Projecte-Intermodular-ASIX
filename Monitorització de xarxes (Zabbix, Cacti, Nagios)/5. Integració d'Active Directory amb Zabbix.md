# Integració d'Active Directory amb Zabbix

## 1. Instal·lació del Rol de Servidor
El primer pas és preparar el servidor Windows Server per actuar com a controlador de domini.

### Instal·lació del Rol AD DS
![Instal·lació ROL](img/AD/Instal·laciò_ROL.png)

En aquesta pantalla del gestor del servidor, seleccionem el rol **"Servicios de dominio de Active Directory"**. Aquest component és el que permetrà gestionar usuaris, equips i polítiques de seguretat de manera centralitzada.

---

## 2. Configuració del Controlador de Domini (Pas a Pas)
Un cop instal·lat el rol, procedim a promocionar el servidor a controlador de domini.

### Pas 1: Selecció de l'operació
![Pas 1](img/AD/DC1.png)

Triem l'opció **"Agregar un nuevo bosque"** i definim el nom del domini arrel com a `aos.cat`. Aquesta és la base de la nostra estructura de directori.

### Pas 2: Opcions del controlador
![Pas 2](img/AD/DC2.png)

Establim els nivells funcionals del bosc i del domini (Windows Server 2016). També definim la **contrasenya de DSRM** (Modo de restauración de servicios de directorio), vital per a la recuperació del sistema.

### Pas 3: Opcions addicionals (NetBIOS)
![Pas 3](img/AD/DC3.png)

El sistema assigna automàticament el nom NetBIOS **AOS**. Aquest nom s'utilitza per a la compatibilitat amb versions anteriors de Windows i per a la identificació a la xarxa local.

### Pas 4: Rutes d'accés
![Pas 4](img/AD/DC4.png)

Definim on s'emmagatzemaran la base de dades (NTDS), els fitxers de registre i la carpeta **SYSVOL**. S'han deixat les rutes per defecte de Windows (`C:\Windows\NTDS` i `C:\Windows\SYSVOL`).

### Pas 5: Revisió d'opcions
![Pas 5](img/AD/DC5.png)

Resum de totes les configuracions seleccionades. És el moment de verificar que el domini, el NetBIOS i els nivells funcionals són correctes abans de procedir amb la instal·lació.

### Pas 6: Comprovació de requisits
![Pas 6](img/AD/DC6.png)

L'assistent verifica que el servidor compleix tots els requisits previs. Un cop apareix el missatge verd de **"Todas las comprobaciones se realizaron correctamente"**, podem fer clic a "Instalar".

### Pas 7: Instal·lació en curs
![Pas 7](img/AD/DC7.png)

Procés d'aplicació de les configuracions, creació de les particions del directori i configuració del servidor DNS integrat.

### Pas 8: Reinici del sistema
![Pas 8](img/AD/DC8.png)

El servidor ha completat la instal·lació dels serveis de domini i requereix un reinici automàtic per configurar els nous rols i el directori.

### Pas 9: Aplicació de configuracions
![Pas 9](img/AD/DC9.png)

Durant el reinici, Windows Server configura els darrers detalls del controlador de domini i els serveis de xarxa necessaris.

### Pas 10: Inici de sessió al domini
![Pas 10](img/AD/DC10.png)

Primer inici de sessió un cop el domini està actiu. Observem que ara ens identifiquem com a **AOS\Administrador**, confirmant que el servidor ja forma part del domini.

### Pas 11: Usuaris i equips d'AD
![Pas 11](img/AD/DC11.png)

Obertura de la consola principal de gestió: **Usuarios y equipos de Active Directory** per començar a administrar el nou directori `aos.cat`.

### Pas 12: Estructura del domini
![Pas 12](img/AD/DC12.png)

Vista inicial de l'arbre del domini `aos.cat` amb les carpetes del sistema (Builtin, Computers, Users, etc.) a punt per ser gestionades.

### Pas 13: Nova Unitat Organitzativa
![Pas 13](img/AD/DC13.png)

Creació d'una **Unitat Organitzativa (UO)** anomenada **Zabbix** per centralitzar tots els objectes (usuaris i equips) relacionats amb el sistema de monitorització.

### Pas 14: Creació d'usuari de monitorització
![Pas 14](img/AD/DC14.png)

Dins de l'UO Zabbix, creem un nou usuari anomenat **zabbix.monitor**. Aquest usuari tindrà els permisos necessaris per realitzar consultes LDAP.

### Pas 15: Contrasenya de l'usuari
![Pas 15](img/AD/DC15.png)

Assignem una contrasenya segura i marquem l'opció **"La contraseña nunca expira"** per garantir que la connexió entre Zabbix i l'AD sigui permanent.

### Pas 16: Resum de l'usuari
![Pas 16](img/AD/DC16.png)

Confirmació final de les dades de l'usuari `zabbix.monitor@aos.cat` abans de guardar l'objecte definitivament al directori.

### Pas 17: Usuari llistat
![Pas 17](img/AD/DC17.png)

L'usuari apareix correctament llistat dins de l'UO Zabbix a la consola de gestió de l'Active Directory.

---

## 3. Integració amb Zabbix i Resolució d'Errors

### Pas 18: Configuració LDAP a Zabbix
![Pas 18](img/AD/DC18.png)

Iniciem la configuració del mètode d'autenticació externa a Zabbix per integrar-lo amb el nostre Active Directory.

### L'Error de PHP LDAP
![Error](img/AD/Error.png)

En intentar activar LDAP, apareix l'error **"Extensión PHP LDAP faltante"**, indicant que el servidor de Zabbix no té instal·lat el mòdul de comunicació necessari.

### Solució - Part 1: Instal·lació
![Solució 1](img/AD/Comanda_solucio1.png)

Executem la comanda d'instal·lació del paquet `php-ldap` al sistema Linux per dotar Zabbix de la capacitat de parlar amb el directori.

### Solució - Part 2: Reinici
![Solució 2](img/AD/Comandes Solucio2.png)

Reiniciem el servidor web Apache o el servei PHP perquè el nou mòdul LDAP s'activi correctament.

### Pas 19: Activació LDAP
![Pas 19](img/AD/DC19.png)

Un cop solucionat l'error de dependències, ja podem activar l'opció d'autenticació LDAP a la interfície de Zabbix.

### Pas 20: Configuració del servidor
![Pas 20](img/AD/DC20.png)

Introduïm les dades del nostre controlador de domini (IP, DN Base, i usuari d'enllaç `zabbix@aos.cat`) per establir la connexió.

### Pas 21: Prova d'autenticació
![Pas 21](img/AD/DC21.png)

Realitzem una prova de login per verificar que Zabbix pot consultar correctament els usuaris de l'Active Directory.

### Pas 22: Configuració de permisos
![Pas 22](img/AD/DC22.png)

Definim com s'assignaran els rols de Zabbix als usuaris que s'autentiquin mitjançant el domini AOS.

### Pas 23: Èxit en la prova
![Pas 23](img/AD/DC23.png)

Zabbix ens confirma el **"Inicio de sesión correcto"**, validant que tota la configuració anterior és funcional.

### Pas 24: Servidor guardat
![Pas 24](img/AD/DC24.png)

El servidor LDAP queda registrat a la llista de mètodes d'autenticació actius de Zabbix.

### Pas 25: Alta d'usuari a Zabbix
![Pas 25](img/AD/DC25.png)

Creem l'usuari `zabbix.monitor` a Zabbix vinculant-lo a l'autenticació LDAP externa.

### Pas 26: Assignació de permisos (Rol)
![Pas 26](img/AD/DC26.png)

Li assignem el rol de **Super administrador** perquè l'usuari tingui control total sobre la monitorització.

### Pas 27: Llista d'usuaris final
![Pas 27](img/AD/DC27.png)

L'usuari ja és visible a la llista global d'usuaris de Zabbix amb l'origen d'autenticació correcte.

### Pas 28: Login amb usuari LDAP
![Pas 28](img/AD/DC28.png)

Fem el login real a Zabbix utilitzant el compte del domini per confirmar que l'usuari pot entrar.

### Pas 29: Panell de control (Dashboard)
![Pas 29](img/AD/DC29.png)

Entrada confirmada. El Dashboard de Zabbix es mostra correctament després de l'autenticació LDAP.

### Pas 30: Monitorització activa
![Pas 30](img/AD/DC30.png)

El sistema ja recull dades del controlador de domini **DC01**, mostrant que l'agent està comunicant correctament.

### Pas 31: Detalls de rendiment
![Pas 31](img/AD/DC31.png)

Gràfics de rendiment en temps real (CPU, RAM) del servidor Windows gestionat pel directori actiu.

### Pas 32: Estat Global del Sistema
![Pas 32](img/AD/DC32.png)

Vista global final que confirma que tota la infraestructura està monitoritzada i el sistema d'autenticació integrat.
