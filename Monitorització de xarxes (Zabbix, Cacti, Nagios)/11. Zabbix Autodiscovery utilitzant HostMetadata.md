
# Auto-registració i Gestió de Hosts a Zabbix

Aquest document descriu el procés per configurar l’auto-registració de hosts Linux a Zabbix, verificar el seu funcionament i gestionar-los (incloent la seva eliminació).

---

## Fase 1: Configuració de l’Agent Zabbix

### Pas 1: Editar la configuració de l’agent

![Editar configuració agent](img/HMD/HM1.png)

```bash
nano /etc/zabbix/zabbix_agentd.conf
```

Aquest fitxer controla la comunicació entre el host i el servidor Zabbix.

---

### Pas 2: Configurar connexió amb el servidor

![Configuració agent](img/HMD/HM2.png)

```bash
ServerActive=192.168.56.101
Hostname=srvlinux01
```

* **ServerActive:** IP del servidor Zabbix
* **Hostname:** Nom del dispositiu a Zabbix

---

### Pas 3: Afegir metadades per autoregistre

![HostMetadata Linux](img/HMD/HM3.png)

```bash
HostMetadata=Linux
```

Permet identificar el tipus de sistema i aplicar accions automàtiques.

---

### Pas 4: Reiniciar el servei

![Restart servei](img/HMD/HM4.png)

```bash
systemctl restart zabbix-agent.service
systemctl status zabbix-agent.service
```

Verificar que el servei està en estat **actiu (running)**.

---

## Fase 2: Configuració de l’Auto-registració

### Pas 1: Crear acció d’autoregistre

![Crear acció](img/HMD/HM8.png)

* Anar a: **Alertes → Accions → Autoregistre**
* Crear una nova acció

---

### Pas 2: Definir condicions

![Condició Linux](img/HMD/HM9.png)

Configurar:

* **Tipus:** Metadatos de equipo
* **Operador:** contiene
* **Valor:** Linux

Això permet que només els hosts Linux s’auto-registrin.

---

### Pas 3: Confirmar condició

![Condició creada](img/HMD/HM10.png)

Verificar que la condició està correctament configurada.

---

## Fase 3: Configuració d’Operacions Automàtiques

### Pas 1: Definir accions

![Operacions](img/HMD/HM11.png)

Afegir:

* Afegir host
* Afegir a grup **Linux servers**
* Assignar plantilles:

  * Linux by Zabbix agent active
  * ICMP Ping
* Activar host

---

### Pas 2: Verificar acció

![Acció creada](img/HMD/HM12.png)

L’acció ha d’estar **activada**.

---

## Fase 4: Verificació del Funcionament

### Pas 1: Hosts descoberts

![Host descobert](img/HMD/HM13.png)

Es mostren els dispositius detectats automàticament a la xarxa.

---

### Pas 2: Host registrat

![Host a Zabbix](img/HMD/HM14.png)

El host apareix a **Recopilació de dades → Equips** amb:

* Estat actiu
* Plantilles assignades
* Monitorització activa

---

## Fase 5: Comprovació amb Logs

### Pas 1: Revisar logs de l’agent

![Logs agent](img/HMD/HM15.png)

```bash
tail -f /var/log/zabbix/zabbix_agentd.log
```

Missatge habitual:

```bash
no active checks on server
```

Aquest missatge és normal abans que el host estigui completament registrat.

---

## Fase 6: Gestió i Eliminació de Hosts

### Pas 1: Accedir a equips

![Llista equips](img/HMD/HM14.png)

* Anar a: **Recopilació de dades → Equips**

---

### Pas 2: Seleccionar el dispositiu

* Marcar el host que es vol eliminar

---

### Pas 3: Eliminar el host

* Fer clic a **Eliminar**
* Confirmar l’acció

---

### Pas 4: Verificar eliminació

* El host desapareix de la llista
* Ja no es monitoritza

---

## Important: Evitar re-registre automàtic

Si l’auto-registració està activa, el host pot tornar a aparèixer.

### Opció 1: Aturar l’agent

```bash
systemctl stop zabbix-agent
```

---

### Opció 2: Eliminar metadades

```bash
nano /etc/zabbix/zabbix_agentd.conf
```

Eliminar o comentar:

```bash
HostMetadata=Linux
```

Reiniciar:

```bash
systemctl restart zabbix-agent
```
