# Auto-registració i Gestió de Hosts a Zabbix
## Fase 1: Configuració de l’Agent Zabbix (Client)

### Pas 1: Accedir i editar el fitxer de configuració de l’agent

![Editar configuració agent](img/HMD/HM1.png)

Accedim al fitxer de configuració de l’agent de Zabbix per modificar els paràmetres necessaris:

```bash
nano /etc/zabbix/zabbix_agentd.conf
```

Aquest fitxer controla la comunicació entre el client i el servidor Zabbix.

---

### Pas 2: Configurar la connexió amb el servidor Zabbix

![Configuració agent](img/HMD/HM2.png)

Dins del fitxer, configurem els següents paràmetres:

```bash
ServerActive=192.168.56.101
Hostname=srvlinux01
```

* **ServerActive**: IP del servidor Zabbix
* **Hostname**: Nom del host (ha de coincidir amb Zabbix)

---

### Pas 3: Afegir metadades per autoregistre

![HostMetadata Linux](img/HMD/HM3.png)

```bash
HostMetadata=Linux
```

Aquest valor permet aplicar regles automàtiques d’auto-registració.

---

### Pas 4: Reiniciar el servei

![Restart servei](img/HMD/HM4.png)

```bash
systemctl restart zabbix-agent.service
systemctl status zabbix-agent.service
```

## Important: Evitar re-registre automàtic

Si no es modifica res, el host tornarà a aparèixer.

### Opció 1: Aturar agent

```bash
systemctl stop zabbix-agent
```

---

### Opció 2: Eliminar metadata

```bash
nano /etc/zabbix/zabbix_agentd.conf
```

Eliminar:

```bash
HostMetadata=Linux
```

---

### Opció 3: Reiniciar servei

```bash
systemctl restart zabbix-agent
```
