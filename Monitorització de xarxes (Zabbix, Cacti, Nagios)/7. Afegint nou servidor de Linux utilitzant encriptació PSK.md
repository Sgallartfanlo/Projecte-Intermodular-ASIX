# Configuracio de Zabbix Agent amb PSK a Linux

Aquest document descriu, pas a pas, com instal.lar i configurar `Zabbix Agent` en un equip Linux i donar-lo d'alta al servidor Zabbix utilitzant xifrat `PSK` (Pre-Shared Key).

La guia esta basada en les captures incloses en aquesta carpeta i serveix com a referencia rapida per repetir el proces en altres hosts.

## Objectiu

- Instal.lar el paquet oficial de `zabbix-agent`.
- Configurar la connexio passiva i activa contra el servidor Zabbix.
- Activar el xifrat `PSK` a l'agent.
- Registrar l'equip al frontal web de Zabbix.

## Dades que cal adaptar

Abans de seguir la guia, substitueix aquests valors pels del teu entorn:

- IP del servidor Zabbix: `192.168.56.101`
- Nom del host Linux: `srvlinux02`
- IP del host Linux: `192.168.56.105`
- Ruta del fitxer de clau PSK: `/opt/encrypted.key`

## Procediment

### 1. Descarregar el repositori i instal.lar l'agent

Es descarrega el paquet de repositori de Zabbix per Ubuntu 24.04, s'instal.la amb `dpkg` i despres s'instal.la `zabbix-agent` amb `apt`.

![Instal.lacio del repositori i del paquet zabbix-agent](img/PSK-Lin/PSK1.png)

### 2. Configurar el servidor per a comprovacions passives

Al fitxer `/etc/zabbix/zabbix_agentd.conf`, cal definir el parametre `Server` amb la IP del servidor Zabbix que podra connectar-se a l'agent.

```ini
Server=192.168.56.101
```

![Configuracio del parametre Server](img/PSK-Lin/PSK4.png)

### 3. Configurar comprovacions actives i el nom del host

Al mateix fitxer, cal indicar:

- `ServerActive` amb la IP del servidor Zabbix
- `Hostname` amb el nom exacte que tindra el host a Zabbix

```ini
ServerActive=192.168.56.101
Hostname=srvlinux02
```

![Configuracio de ServerActive i Hostname](img/PSK-Lin/PSK5.png)

### 4. Activar xifrat PSK a l'agent

Configura l'agent per usar `PSK` tant en la connexio sortint com en les connexions entrants.

```ini
TLSConnect=psk
TLSAccept=psk
```

![Activacio de TLS amb PSK](img/PSK-Lin/PSK6.png)

### 5. Definir la identitat PSK i el fitxer de clau

Assigna un identificador PSK i indica la ruta del fitxer on es guardara la clau compartida.

```ini
TLSPSKIdentity=srvlinux02
TLSPSKFile=/opt/encrypted.key
```

![Identitat PSK i ruta del fitxer de clau](img/PSK-Lin/PSK7.png)

### 6. Generar la clau PSK

Es genera una clau hexadecimal aleatoria de 32 bytes amb `openssl`.

```bash
openssl rand -hex 32
```

![Generacio de la clau PSK](img/PSK-Lin/PSK8.png)

### 7. Guardar la clau en un fitxer

La clau generada s'ha de desar al fitxer definit a `TLSPSKFile`, per exemple:

```bash
nano /opt/encrypted.key
```

Despres de desar-la, es recomana ajustar permisos per protegir-la:

```bash
chmod 600 /opt/encrypted.key
chown zabbix:zabbix /opt/encrypted.key
```

![Clau PSK desada al fitxer /opt/encrypted.key](img/PSK-Lin/PSK9.png)

### 8. Habilitar i reiniciar el servei

Un cop aplicada la configuracio, cal habilitar el servei a l'arrencada i reiniciar-lo.

```bash
systemctl enable zabbix-agent.service
systemctl restart zabbix-agent.service
```

![Habilitacio del servei](img/PSK-Lin/PSK3.png)

![Reinici del servei](img/PSK-Lin/PSK10.png)

### 9. Verificar l'estat de l'agent

Comprova que el servei esta actiu i en execucio:

```bash
systemctl status zabbix-agent.service
```

![Verificacio del servei zabbix-agent](img/PSK-Lin/PSK2.png)

### 10. Crear el host al frontal web de Zabbix

Al panell web, ves a `Recopilacion de datos > Equipos` i prem `Crear equipo`.

![Pantalla d'equips a Zabbix](img/PSK-Lin/PSK11.png)

### 11. Omplir la configuracio basica del host

En crear el nou host:

- `Nombre de equipo`: `srvlinux02`
- Plantilla: `Linux by Zabbix agent active`
- Grup: `Linux servers`
- Interficie d'agent: IP `192.168.56.105`, port `10050`

El nom del host ha de coincidir amb el valor configurat a `Hostname`.

![Configuracio basica del nou host](img/PSK-Lin/PSK12.png)

### 12. Configurar el xifrat PSK al host de Zabbix

A la pestanya `Cifrado`:

- Selecciona `PSK` per a les connexions al host
- Marca `PSK` per a les connexions des del host
- Defineix `Identidad PSK` amb el mateix valor que `TLSPSKIdentity`
- Enganxa la mateixa clau generada al camp `PSK`

La identitat i la clau han de coincidir exactament entre servidor i agent.

![Configuracio PSK al frontal de Zabbix](img/PSK-Lin/PSK13.png)

## Resum de configuracio minima

El bloc essencial de `/etc/zabbix/zabbix_agentd.conf` queda aixi:

```ini
Server=192.168.56.101
ServerActive=192.168.56.101
Hostname=srvlinux02
TLSConnect=psk
TLSAccept=psk
TLSPSKIdentity=srvlinux02
TLSPSKFile=/opt/encrypted.key
```

## Recomanacions

- Protegeix el fitxer PSK amb permisos restrictius.
- Verifica que el port `10050/TCP` sigui accessible des del servidor Zabbix.
- Comprova que el `Hostname` de l'agent i el del frontal web coincideixen exactament.
- Si canvies la clau PSK, actualitza-la tant a l'agent com al servidor.
